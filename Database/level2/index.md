## インデックスの作成
```sql
mysql> alter table employees add index birth_date_index(birth_date);

Records: 0  Duplicates: 0  Warnings: 0
```

## ベストプラクティス
where句、order by句、joinの結合条件に頻繁する列はインデックスの効果が得られやすい
インデックスによりディスク容量を圧迫してしまう、データ書き換え時にオーバーヘッドが発生する可能性というデメリットもある。
Explain plan文を使用することでindexを設定した場合にどのくらい高速化できるかを事前に調べることができるため、
必要に応じて使用すべき。
